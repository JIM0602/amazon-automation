# 生成时间: 2026-03-31
# 验证结果: PASS

## 模块：src/utils/killswitch.py

### 函数签名
- is_stopped() -> bool
- activate_stop(reason: str, triggered_by: str) -> None
- deactivate_stop(triggered_by: str) -> None
- check_killswitch() -> Callable（装饰器工厂）
- SystemStoppedError（自定义异常，含 reason 属性）

### 设计特点
1. is_stopped() 每次查询 DB（不缓存），保证实时生效
   - 支持 JSON 字段中的字符串 "true"/"false" 和布尔值 True/False
   - DB 错误时安全降级返回 False
2. activate_stop() 操作流程：
   - 写入 system_config（emergency_stop="true" 及元数据）
   - 调用 _pause_all_jobs() 暂停所有 APScheduler 任务
   - 记录 audit_log（action="emergency_stop_activated"）
   - 发送飞书通知（如可用）
3. deactivate_stop() 操作流程：
   - 读取停机前状态信息（用于 audit log pre_state）
   - 更新 system_config（emergency_stop="false"）
   - 调用 _resume_all_jobs() 恢复所有任务
   - 记录 audit_log（action="emergency_stop_deactivated"）
4. check_killswitch() 装饰器：
   - 每次在函数执行前调用 is_stopped() 检查状态
   - 停机时 raise SystemStoppedError（包含函数名和停机原因）
   - 使用 functools.wraps 保留原函数元数据

### system_config 表使用的 key
- emergency_stop: "true"/"false"
- emergency_stop_reason: 停机原因
- emergency_stop_triggered_by: 触发者
- emergency_stop_activated_at: 激活时间（ISO 8601）
