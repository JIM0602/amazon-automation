# Amazon SP-API 开发者申请指南

## 申请步骤

1. 注册 Amazon Developer Central 账号，并绑定卖家/开发者主体。
2. 在 Developer Central 中创建应用，填写应用名称、用途说明、回调地址等基础信息。
3. 配置 IAM/安全相关信息，生成并保存 Access Key、Secret Key、Refresh Token 所需的授权链路。
4. 关联目标店铺与 Marketplace（美国站），完成授权同意流程。
5. 等待 Amazon 审核，补充必要材料后即可逐步接入 SP-API。

> 说明：Phase 1 完全使用 Mock 数据，不做任何真实 SP-API 写入操作；SP-API 申请与联调作为 Phase 2 的准备工作。

## 所需材料

- 公司主体信息：公司名称、地址、联系人、营业执照/注册信息。
- 税务与合规信息：VAT/税号（如适用）、退款与开票信息。
- 品牌信息：品牌名、商标/品牌备案信息、站点信息。
- 店铺信息：Seller Central 账号、店铺 ID、目标 Marketplace ID。
- 技术信息：回调地址、应用说明、权限范围说明、数据使用场景。

## 常见问题

### 1. 为什么会被拒绝？

- 主体信息不完整或与店铺信息不一致。
- 应用说明不清晰，无法说明调用 SP-API 的真实业务场景。
- 权限申请过多，与当前需求不匹配。
- 安全/回调配置不正确，或无法完成授权验证。

### 2. 如何提高通过率？

- 先用最小权限申请，按业务逐步扩展。
- 保持公司、品牌、店铺信息一致。
- 提前准备清晰的集成说明和数据流说明。

### 3. 申请周期多久？

- 通常需要 2-4 周，期间可先使用 Mock 数据推进 Phase 1 开发与测试。
