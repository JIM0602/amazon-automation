# 08-常见问题 FAQ：为您解答运营疑惑

以下是 PUDIWIND AI 助手在使用过程中最常见的问题及其解答。

---

## 🙋 常见问题 Q&A

### 1. 为什么我发的指令机器人没理我？
- **可能原因**：指令不匹配或包含了机器人不认识的词汇。
- **解决步骤**：请确保您的指令以 `?`、`日报` 或 `选品` 开头。例如，发送 `?怎么申诉` 而不是简单的 `申诉`。

### 2. "选品分析" 为什么需要这么久？
- **解答**：系统在接到指令后，需要实时从亚马逊抓取前 20 名竞品的数据，并调用 AI 专家模型进行逻辑推理，这个过程大约需要 **3-5 分钟**。请在发送指令后耐心等待机器人回复。

### 3. 我可以修改已经生成的 "日报" 吗？
- **解答**：日报展示的是昨日发生的历史客观数据，系统不支持手动修改。如果您发现数据与亚马逊后台不符，请联系技术团队排查数据同步是否正常。

### 4. 机器人可以帮我直接修改亚马逊后台的价格吗？
- **解答**：出于安全考虑，目前的 AI 助手仅提供**数据分析和决策建议**，不具备直接操作亚马逊后台（如调价、发货）的权限。所有操作均需人工在亚马逊卖家中心完成。

### 5. 如果我在群里 @机器人，别人能看到回复吗？
- **解答**：是的。在群聊中，机器人的回复对群内所有成员可见。如果您想进行私密的选品分析，请前往机器人的单聊窗口。

### 6. 系统显示的 "库存告警" 准吗？
- **解答**：准。系统通过亚马逊官方 API 接口（FBA 接口）实时获取可售库存数据。一旦低于预设阈值（如 20 件），系统会自动发出 🔴 红色告警。

### 7. 我可以上传自己的运营文档到资料库吗？
- **解答**：目前资料库由技术运营团队管理。如果您有优质的 SOP（标准作业程序），请发送给部门主管，由技术人员统一导入资料库，这样全团队的 AI 助手都会变得更聪明。

### 8. 如果系统提示 "紧急停机" 怎么办？
- **解答**：这通常意味着系统检测到了潜在的数据风险或程序异常。此时请停止发送指令，等待技术负责人在群内发布系统恢复通知。

### 9. 怎么才能让 AI 给出的选品建议更精准？
- **技巧**：在指令后加上具体的关键词。例如发送 `选品分析 宠物自动饮水机` 比只发送 `选品分析 宠物用品` 能得到更具实操价值的建议。

### 10. 机器人可以识别语音消息吗？
- **解答**：目前 AI 助手仅支持**文字指令**。发送语音消息将导致机器人返回 "无法识别" 的提示。

## 👨‍💻 技术人员参考 (Technical Reference)

> **注**：本部分仅供具备技术背景的开发或运维人员参考，运营人员可忽略。

- **消息路由**：核心逻辑位于 `src/feishu/command_router.py`。
- **Agent 工作流**：`SelectionAgent` 使用 LangGraph (见 `src/agents/selection_agent/agent.py`)，每日报告由 `DailyReportAgent` 驱动。
- **紧急停机**：状态存储在 `system_config` 表的 `emergency_stop` 键中，由 `src/utils/killswitch.py` 装饰器控制执行流程。
- **数据流**：所有写操作必须记录到 `audit_logs` 表，多维表格同步使用飞书 Bitable v1 REST API。
- **错误排查**：若机器人无响应且未停机，请检查飞书回调 Endpoint 的服务器日志及 PostgreSQL 数据库连接。
