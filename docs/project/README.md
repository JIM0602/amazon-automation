# 项目文档基线

本目录用于存放当前仓库的正式项目文档基线，服务于后续持续开发、范围控制与状态同步。

## 文档入口
- [PRD](./prd.md) — 当前项目目标、范围、角色边界与优先级
- [架构说明](./architecture.md) — 当前系统结构、关键模块、数据流与安全约束
- [项目状态](./status.md) — 当前完成度、已知缺口、最近完成事项与下一步计划

## 使用规则
- 功能范围、页面边界、角色能力发生变化时，更新 `prd.md`
- 模块边界、数据流、鉴权方式、安全约束发生变化时，更新 `architecture.md`
- 迭代结果、当前缺口、下一步排序发生变化时，更新 `status.md`

## 约束
- 本目录记录的是当前仓库的正式事实基线，不是历史构思草稿
- `README.md`、`PROJECT_DOCUMENTATION.md`、`构思.md`、`构思补充.md` 仍可作为背景参考，但以本目录为后续开发对齐基线
- 正式项目文档统一放在 `docs/project/`，不与 `docs/ai-agents/**` 等第三方或历史资料混放
